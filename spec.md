# Refra - チームリファレンス共有ファイルビューワー 仕様書

## 1. プロダクト概要

### 1.1 目的
社内チーム（20〜30人規模）がアプリ開発時に「このUIいいよね」「この演出がいいよね」「このモデルいいよね」といったリファレンス素材（画像・動画）を共有・閲覧・比較・コメントするためのWebアプリケーション。

### 1.2 基本方針
- **完全サーバーレス**: DBサーバーは立てない。メタデータはS3上のJSONファイルで管理
- **フロント中心処理**: サムネイル生成、動画クリップ切り出し等はすべてブラウザ側で処理
- **MVP優先**: 段階的に機能追加。URLからの動画ダウンロードはMVPでは手動運用
- **セキュリティは軽量**: アクセスポイント非公開＋Cognito認証で十分

### 1.3 アプリ名
**Refra**（Reference + Viewの造語）

---

## 2. アーキテクチャ

### 2.1 全体構成

```
[ブラウザ (Vite + React + TS)]
    │
    ├── 認証: AWS Cognito (Google Workspace連携、ドメイン制限)
    │
    └── ストレージ: AWS S3
          ├── /assets/          ... 画像・動画ファイル本体
          ├── /thumbnails/      ... サムネイル画像
          ├── /scenes/          ... 動画クリップ（シーン）
          ├── /meta/index.json  ... 軽量インデックス（一覧用）
          └── /meta/{assetId}.json ... 個別メタデータ（コメント等含む）
```

### 2.2 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロントエンド | Vite + React + TypeScript |
| UIライブラリ | shadcn/ui (Tailwind CSS) |
| ローカルDB | IndexedDB (Dexie.js) |
| ストレージ | AWS S3（本番）/ LocalStack（ローカル開発） |
| 認証 | AWS Cognito + Google Workspace（本番）/ なし（ローカル） |
| 動画処理 | ffmpeg.wasm（ブラウザ内） |
| ホスティング | S3 + CloudFront（静的サイト配信） |

### 2.3 ローカル開発環境
- S3モック: **LocalStack**（Docker Compose）
- 認証: 無効化（開発モードフラグ）
- ストレージアクセスはインターフェースで抽象化し、LocalStack ↔ S3を切り替え可能にする

## 3. データモデル

### 3.1 用語定義

| 用語 | 説明 |
|------|------|
| **アセット (Asset)** | アップロードされた元ファイル（画像 or 動画） |
| **シーン (Scene)** | 動画アセットから切り出された数秒のクリップ。画像の場合はアセットとシーンが1:1 |

### 3.2 メタデータ構造

#### ハイブリッド方式
- **インデックスファイル** (`/meta/index.json`): 一覧表示用の軽量データ
- **個別メタデータ** (`/meta/{assetId}.json`): 詳細情報（コメント等）

競合を最小化するため、書き込みは個別メタデータJSONに対して行う。インデックスファイルはアップロード/削除時に更新する。

#### インデックスファイル (`/meta/index.json`)
```json
{
  "version": 1,
  "updatedAt": "2025-03-14T12:00:00Z",
  "assets": [
    {
      "id": "uuid-v4",
      "name": "表示名",
      "type": "image" | "video",
      "thumbnailPath": "/thumbnails/{id}.webp",
      "folderId": "folder-uuid | null",
      "tags": ["UI", "演出", "3Dモデル"],
      "createdBy": "Googleアカウント名",
      "createdAt": "2025-03-14T12:00:00Z",
      "updatedAt": "2025-03-14T12:00:00Z"
    }
  ],
  "folders": [
    {
      "id": "folder-uuid",
      "name": "フォルダ名",
      "parentId": "parent-folder-uuid | null",
      "createdBy": "Googleアカウント名",
      "createdAt": "2025-03-14T12:00:00Z"
    }
  ]
}
```

#### 個別メタデータ (`/meta/{assetId}.json`)
```json
{
  "id": "uuid-v4",
  "name": "表示名",
  "type": "image" | "video",
  "originalPath": "/assets/{id}.{ext}",
  "thumbnailPath": "/thumbnails/{id}.webp",
  "folderId": "folder-uuid | null",
  "tags": ["UI", "演出"],
  "sourceUrl": "https://youtube.com/... | null",
  "sourceUrlMeta": {
    "title": "動画タイトル",
    "channel": "チャンネル名",
    "url": "元URL"
  },
  "resolution": { "width": 1920, "height": 1080 },
  "duration": 3600,
  "scenes": [
    {
      "id": "scene-uuid",
      "startTime": 120.5,
      "endTime": 125.0,
      "clipPath": "/scenes/{assetId}/{sceneId}.webm",
      "thumbnailPath": "/thumbnails/scenes/{sceneId}.webp",
      "createdBy": "Googleアカウント名",
      "createdAt": "2025-03-14T12:00:00Z"
    }
  ],
  "comments": [
    {
      "id": "comment-uuid",
      "text": "ここのモーションがいいよね",
      "author": "Googleアカウント名",
      "timestamp": null,
      "createdAt": "2025-03-14T12:00:00Z"
    },
    {
      "id": "comment-uuid",
      "text": "画面切り替えのタイミングが絶妙",
      "author": "Googleアカウント名",
      "timestamp": 65.3,
      "createdAt": "2025-03-14T12:00:00Z"
    }
  ],
  "createdBy": "Googleアカウント名",
  "createdAt": "2025-03-14T12:00:00Z",
  "updatedAt": "2025-03-14T12:00:00Z"
}
```

#### コメントの種類（動画の場合）
- `timestamp: null` → 動画全体に対するコメント
- `timestamp: 65.3` → 動画の65.3秒地点に紐づくコメント（シークバー上にマーカー表示）

#### 画像の場合
- アセットとシーンは1:1。`scenes`配列にはアセット全体を示す1エントリのみ
- `duration`は`null`

### 3.3 ローカル設定（IndexedDB）
設定はクラウドに上げない。ローカルのみに保持。
```json
{
  "theme": "dark" | "light",
  "thumbnailSize": 200,
  "locale": "ja"
}
```

### 3.4 メタデータ競合処理
- 書き込み時: S3のETag（楽観的ロック）を利用して `If-Match` ヘッダーで更新
- 競合発生時: 自動リトライ（最新を再取得→マージ→再PUT）
- 20〜30人規模で、同じアセットに同時書き込みする頻度は低いため、この方式で十分

## 4. 画面構成・UI仕様

### 4.1 UIの基本方針
- **日本語のみ**（i18n不要）
- **ダーク/ライト切り替え対応**（映像確認用途のため背景色は重要）
- デフォルトはダークモード

### 4.2 画面一覧

#### (A) サムネイル一覧画面（メイン画面）

**レイアウト:**
```
┌──────────────────────────────────────────────────┐
│ [サイドバー]         [サムネイルグリッド]          │
│ ┌──────────┐  ┌──────────────────────────────┐   │
│ │ フォルダ   │  │ [サイズスライダー] [比較モード]│   │
│ │ ツリー     │  │ ┌────┐┌────┐┌────┐┌────┐   │   │
│ │           │  │ │    ││    ││    ││    │   │   │
│ │ ──────── │  │ │サムネ││サムネ││サムネ││サムネ│   │   │
│ │ タグ      │  │ │    ││    ││    ││    │   │   │
│ │ フィルタ  │  │ └────┘└────┘└────┘└────┘   │   │
│ │           │  │ ┌────┐┌────┐┌────┐┌────┐   │   │
│ │           │  │ │    ││    ││    ││    │   │   │
│ └──────────┘  │ └────┘└────┘└────┘└────┘   │   │
│               └──────────────────────────────┘   │
└──────────────────────────────────────────────────┘
```

**機能:**
- サムネイルのサイズはスライダーで可変（デフォルト約200px）
- グリッドはレスポンシブ。サイズに応じて列数が自動調整
- 各サムネイルにはアセット名、タグがオーバーレイ表示
- 動画の場合はサムネイル左下に再生時間を表示
- **大量表示優先**: 一般的なファイルビューワーより表示密度を高めに
- 仮想スクロール（react-window等）で大量アセットに対応

**インタラクション:**
- **ホバー**: プレビューがポップアップで拡大表示（画像はそのまま拡大、動画はプレビュー再生）
- **クリック**: 詳細画面(B)に遷移
- **右クリック**: コンテキストメニュー（フォルダ移動、タグ編集、削除）

**サイドバー:**
- フォルダツリー（ネスト対応、ユーザーが自由に作成・移動・リネーム可能）
- タグフィルター（AND/OR切り替え）
- 検索バー（アセット名、タグで検索）

#### (B) 詳細画面（個別アセット）

**画像の場合:**
```
┌──────────────────────────────────────────────────┐
│ [← 戻る]  アセット名  [タグ編集] [削除]           │
│ ┌────────────────────────┐  ┌──────────────────┐ │
│ │                        │  │ コメント一覧      │ │
│ │    オリジナル画像表示    │  │                  │ │
│ │    (ズーム/パン可能)    │  │ > ユーザーA:      │ │
│ │                        │  │   この配色いい    │ │
│ │                        │  │ > ユーザーB:      │ │
│ │                        │  │   影の付け方参考  │ │
│ │                        │  │                  │ │
│ └────────────────────────┘  │ [コメント入力]    │ │
│ 引用元: https://...         └──────────────────┘ │
└──────────────────────────────────────────────────┘
```

**動画の場合:**
```
┌──────────────────────────────────────────────────┐
│ [← 戻る]  アセット名  [タグ編集] [削除]           │
│ ┌────────────────────────────────────────────┐   │
│ │              動画プレイヤー                  │   │
│ │                                            │   │
│ └────────────────────────────────────────────┘   │
│ [▶] ──●──△──△────●───────── 1:23:45 / 2:30:00  │
│       ↑コメントマーカー  ↑シーン範囲              │
│                                                  │
│ ┌─ シーン一覧 ──────┐  ┌── コメント ────────────┐│
│ │ [+ シーン追加]     │  │ 全体コメント:          ││
│ │ ┌────┐ 0:02:00-   │  │  > ユーザーA: ...      ││
│ │ │    │ 0:02:05    │  │ タイムスタンプ:         ││
│ │ └────┘            │  │  > 1:23 ユーザーB: ... ││
│ │ ┌────┐ 0:15:30-   │  │ [コメント入力]         ││

│ │ └────┘            │                            │
│ └────────────────────┘                            │
└──────────────────────────────────────────────────┘
```

**動画詳細の機能:**
- シークバー上にコメントマーカー（タイムスタンプ付きコメントの位置）を表示
- シークバー上にシーン範囲をハイライト表示
- 「シーン追加」ボタンで開始/終了時間を指定 → ffmpeg.wasmでブラウザ内切り出し
- コメントは「全体コメント」と「タイムスタンプ付きコメント」の2種類
- タイムスタンプ付きコメントをクリックすると該当秒にジャンプ
- 引用元URLのメタ情報（タイトル、チャンネル名）を表示

#### (C) 比較モード

**フロー:**
1. メイン画面の「比較モード」トグルをON
2. 画面右側にドロップゾーンカラムが出現
3. サムネイルをD&Dでカラムに追加（最大4つ）
4. 「View」ボタンで比較画面に遷移

**比較画面レイアウト（選択数に応じて動的）:**
- 1つ: 全画面
- 2つ: 左右50:50分割
- 3つ: 左50% + 右上25%・右下25%
- 4つ: 2×2グリッド

**比較画面の機能:**
- コメント: **表示のみ**（書き込みは個別詳細画面で行う）
- 動画再生: **同期/個別の切り替えが可能**
  - 同期モード: 全動画が一斉にplay/pause、シークも連動
  - 個別モード: それぞれ独立して操作
- 各パネルにアセット名を表示
- ESCまたは「閉じる」で一覧画面に戻る

#### (D) アップロード画面

**対応方法:**
- ドラッグ&ドロップ
- ファイル選択ダイアログ
- 複数ファイル同時アップロード対応

**アップロードフロー（画像）:**
1. ファイル選択/D&D
2. ブラウザ側でサムネイル生成（Canvas API、WebP形式、200px幅）
3. アセット名入力、タグ付け、フォルダ選択
4. オリジナル + サムネイルをS3にアップロード
5. メタデータJSON作成 → S3にPUT
6. インデックスJSON更新（ETagチェック付き）

**アップロードフロー（動画）:**
1. ファイル選択/D&D
2. ブラウザ側で特定フレームからサムネイル生成（video要素 + Canvas API）
3. アセット名入力、タグ付け、フォルダ選択
4. オリジナル + サムネイルをS3にアップロード（S3マルチパートアップロード使用）
5. アップロード後、シーン切り出しUIを表示
   - 開始時間/終了時間を指定
   - ffmpeg.wasmでクリップ生成 → S3にアップロード
   - シーンは後からいつでも追加可能
6. メタデータJSON作成 → S3にPUT
7. インデックスJSON更新

**アップロード制限:**
- 1ファイルあたりの上限: **無制限**（ストレージ代は許容）
- 大きいファイルはS3マルチパートアップロードを使用
- プログレスバー表示

**引用元URL入力:**
- 任意のURL入力フィールド
- MVPではURLからの自動ダウンロードは非対応（手動でダウンロード→アップロードの運用）
- 将来的にFargate等で自動ダウンロード機能を追加可能

#### (E) 設定画面

- テーマ切り替え（ダーク/ライト）
- サムネイルデフォルトサイズ
- 設定はローカルのみ保持（IndexedDB）
- クラウド同期なし

### 4.3 削除機能
- **誰でも削除可能**（権限管理なし）
- 削除時は確認ダイアログを表示
- 削除対象: S3上のファイル（オリジナル、サムネイル、シーンクリップ）+ メタデータJSON + インデックスからの除去

## 5. ファイル処理仕様

### 5.1 対応フォーマット

**画像:**
- 入力: JPEG, PNG, WebP, GIF
- サムネイル: WebP（200px幅ベース、アスペクト比維持）
- 解像度: アップロード時にフルHD（1920×1080）以上ならリサイズするかは任意。オリジナルはそのまま保持

**動画:**
- 入力: MP4, WebM, MOV
- シーン（クリップ）保存形式: **WebM（VP9、音声なし）** ← 最軽量かつブラウザ互換性良好
- サムネイル: WebP
- フレームレート: 元動画のフレームレートを維持（リファレンス用途なのでフレーム数が重要）
- 解像度: フルHD上限。メタ情報としてオリジナル解像度を保持

### 5.2 サムネイル生成（フロント処理）

**画像:**
```
Canvas API → drawImage (リサイズ) → toBlob('image/webp', 0.8)
```

**動画:**
```
video要素にsrc設定 → seekToTime(1秒目) → canvasにdrawImage → toBlob('image/webp', 0.8)
```

### 5.3 シーン切り出し（フロント処理）

```
ffmpeg.wasm ロード
→ 元動画ファイルをffmpeg仮想FSに書き込み
→ ffmpeg -i input -ss {start} -to {end} -an -c:v libvpx-vp9 -b:v 1M output.webm
→ 出力ファイルをS3にアップロード
```

注意: 大きな動画ファイル（数GB）の場合、ブラウザのメモリが不足する可能性あり。
対策: ユーザーに「大きなファイルの場合は時間がかかります」と表示。将来的にはFargate対応。

## 6. S3バケット構成

```
refra-{env}/
├── assets/
│   ├── {assetId}.jpg
│   ├── {assetId}.mp4
│   └── ...
├── thumbnails/
│   ├── {assetId}.webp
│   └── scenes/
│       └── {sceneId}.webp
├── scenes/
│   └── {assetId}/
│       └── {sceneId}.webm
└── meta/
    ├── index.json
    └── {assetId}.json
```

### 6.1 S3設定
- CORS: フロントエンドのオリジンを許可
- バケットポリシー: Cognito認証ユーザーのみアクセス可能
- マルチパートアップロード: 大容量動画ファイル用に有効化

## 7. 認証（本番環境）

### 7.1 構成
- **AWS Cognito User Pool** + Google Workspaceフェデレーション
- ドメイン制限: 特定のGoogle Workspaceドメインのみ許可
- Cognito Identity Pool → S3バケットへの一時クレデンシャル発行

### 7.2 フロー
1. ユーザーがアプリにアクセス
2. Cognito Hosted UIでGoogleログイン
3. Google Workspaceドメインチェック
4. Cognito IDトークン取得
5. Identity Poolで一時的なAWSクレデンシャル取得
6. S3に直接アクセス（ブラウザから）

### 7.3 ローカル開発時
- 認証スキップ（環境変数 `VITE_AUTH_DISABLED=true`）
- LocalStackのS3に直接アクセス

## 9. 技術的判断ログ

以下は仕様策定時に検討・決定した事項のログ。実装時に「なぜこうなったか」を参照するためのもの。

| 項目 | 決定 | 理由 |
|------|------|------|
| メタデータ管理 | ハイブリッド（インデックス + 個別JSON） | 競合最小化。1つの大きなJSONだと20-30人で競合しやすい |
| ローカルS3モック | LocalStack | MinIOは2025年12月にメンテナンスモード→2026年2月アーカイブ済み。開発モック用途ならLocalStackが最も安定 |
| 動画クリップ形式 | WebM (VP9, 音声なし) | 最軽量かつブラウザ互換性良好。音声不要のリファレンス用途に最適 |
| 動画処理 | ブラウザ内 (ffmpeg.wasm) | Lambda/Fargateなしで完全サーバーレス維持。MVPではURLダウンロードは手動運用 |
| フレームワーク | Vite + React + TS | 軽量SPA。Next.jsほどのSSR/ルーティングは不要 |
| 本番ストレージ | AWS S3 | Cognito含めAWSで統一。R2はエグレス無料だが認証周りの統一性を優先 |
| 認証 | Cognito + Google Workspace | サーバーレス。ドメイン制限で社内限定 |
| UIテーマ | ダーク/ライト切り替え | 映像確認用途で背景色が重要 |
| 削除権限 | 誰でも削除可能 | 社内ツール、20-30人規模で権限管理は過剰 |
| アップロード上限 | 無制限 | VTuber配信動画（数GB）のアップロードに対応。ストレージ代は許容 |

## 11. テスト戦略

### 11.1 基本方針: TDD（テスト駆動開発）

**すべての実装はTDDで行う。** 機能実装の前にテストを書き、Red → Green → Refactorのサイクルを回す。

LLM（Claude Code）による開発だからこそテストが重要:
- コード生成時の意図しない破壊を即座に検知
- リファクタリング時の安全ネット
- 仕様のドキュメントとしての役割

### 11.2 テストツール

| 種別 | ツール | 用途 |
|------|--------|------|
| ユニットテスト | Vitest | サービス層、ユーティリティ、ストア |
| コンポーネントテスト | Vitest + React Testing Library | UIコンポーネントの振る舞い |
| E2Eテスト | Playwright | 画面遷移、ユーザーフロー、ビジュアル検証 |

### 11.3 ユニットテスト（Vitest）

**TDDサイクルで最初に書くテスト。** 各サービス・ユーティリティの実装前にテストを書く。

対象と例:
- `services/metadata.ts`: メタデータのCRUD、インデックス更新、競合時リトライ
- `services/storage.ts`: S3アップロード/ダウンロード/削除（LocalStackに対して実行）
- `services/thumbnail.ts`: サムネイル生成のパラメータ検証（Canvas APIのモック）
- `types/index.ts`: 型バリデーション（zod等でランタイムバリデーションする場合）
- フォルダのツリー構造操作（作成、移動、リネーム、ネスト）
- タグのフィルタリングロジック（AND/OR）
- コメントのソート、タイムスタンプ付きコメントのフィルタ

```typescript
// 例: metadata.test.ts（テストを先に書く）
describe('MetadataService', () => {
  it('アセット作成時にインデックスにエントリが追加される', async () => {
    const asset = createTestAsset({ name: 'テスト画像', type: 'image' });
    await metadataService.createAsset(asset);
    const index = await metadataService.getIndex();
    expect(index.assets).toContainEqual(expect.objectContaining({ id: asset.id }));
  });

  it('ETag不一致時にリトライしてマージする', async () => {
    // ... 競合シナリオのテスト
  });
});
```

### 11.4 コンポーネントテスト（React Testing Library）

UIコンポーネントの振る舞いをテスト。DOM操作やイベントの結果を検証。

対象と例:
- `ThumbnailGrid`: サイズスライダー変更で列数が変わること
- `FolderTree`: フォルダ展開/折りたたみ、D&Dによる移動
- `CommentPanel`: コメント追加、表示順
- `UploadDropzone`: ファイルD&D受付、バリデーション
- `VideoPlayer`: 再生/停止、シークバー操作
- `CompareDropZone`: D&Dでアセット追加、最大4つ制限

### 11.5 E2Eテスト（Playwright）

**各Phase・各機能実装の区切りでE2Eテストを実行。** スクリーンショットを撮って視覚的に検証する。

#### E2Eテストの実行タイミング
- **機能単位**: 新しい画面や主要機能が実装できたタイミングで必ず実行
- **Phase区切り**: 各Phaseの完了時にフルスイートを実行
- **リファクタリング後**: 大きなリファクタ後にリグレッションチェック

#### スクリーンショット検証
Claude Codeでの開発時、Playwrightのスクリーンショット機能でUIの状態を視覚的に確認する:

```typescript
// 例: スクリーンショットを撮って視覚検証
test('サムネイル一覧が正しく表示される', async ({ page }) => {
  await page.goto('/');
  // テストデータのアセットが表示されること
  await expect(page.getByTestId('thumbnail-grid')).toBeVisible();
  await page.screenshot({ path: 'screenshots/gallery-default.png', fullPage: true });
});

test('ダークモードとライトモードの切り替え', async ({ page }) => {
  await page.goto('/');
  await page.screenshot({ path: 'screenshots/theme-dark.png' });
  await page.getByTestId('theme-toggle').click();
  await page.screenshot({ path: 'screenshots/theme-light.png' });
});
```

#### E2Eテストシナリオ

**Phase 1完了時:**
- [ ] 画像アップロード → サムネイル一覧に表示される
- [ ] サムネイルサイズスライダーが機能する
- [ ] ホバーでプレビュー拡大される
- [ ] クリックで詳細画面に遷移する
- [ ] フォルダ作成 → フォルダへのアセット移動
- [ ] タグ付与 → タグフィルタリング
- [ ] コメント追加 → 表示される
- [ ] アセット削除 → 一覧から消える
- [ ] ダーク/ライトテーマ切り替え
- [ ] 各画面のスクリーンショット取得（ビジュアル検証）

**Phase 2完了時:**
- [ ] 動画アップロード → プレイヤーで再生ã → シーン一覧に表示される
- [ ] タイムスタンプ付きコメント → シークバーにマーカー表示
- [ ] コメントクリック → 該当秒にジャンプ

**Phase 3完了時:**
- [ ] 比較モードON → ドロップゾーン表示
- [ ] D&Dでアセット追加（最大4つ）
- [ ] View → 選択数に応じたレイアウト（1/2/3/4分割）
- [ ] 動画同期再生/個別再生の切り替え
- [ ] 比較画面でコメント表示（読み取りのみ）

#### テタ
E2Eテスト用にシードデータ（テスト用画像・動画・メタデータ）をLocalStackに投入するスクリプトを用意する:

```
└── seed-test-data.ts        # テスト用アセット・メタデータ投入
```
